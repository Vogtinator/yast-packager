YaST2 - Package-Installation

Overview

Package-installation is a module for installing and deleting packages in the running system. There are two kinds of using this module:

  • Manually package-selection and package-installation
  • Automatically package-selection and package-installation

Manually package-selection

    In this case the workflow is started with the package-single-selection. The user can select or deleselct packages which have to be installed or deleted. After he has confirmed the selection the packages will be installed or deleted and SuSEConfig will be started.

    Module call:

    yast2 sw_single

Automatically package-selection

    In this case the workflow is started with a pre-selected number of packages which have been defined in an ascii-file ( YCP-format ). This packages will be installed or deleted automatically after resolving the dependencies. SuSEConfig will be started at least.

    Module call:

    yast2 sw_single '"filename"'

    filename is path on an ascii-file, where packages are defined for installation or deinstalltion.

    Example of description file:

    $["userInput":true, "install":["tuxeyes","xroach"], "delete":["xlock"]]

    userInput: If userInput is true the user can change the packages which have been pre-selected by calling the module package-single-selection. If it is false package-single-selection will be called if a package-confilct has been occured ( default is false ).

    install: Defines a list of packages, which have to be installed.

    delete: Defines a list of packages, which have to be deleted.
